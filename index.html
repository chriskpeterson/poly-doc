Audio Configuration & Tuning

Configure advanced audio processing, external DSPs, and microphone arrays.

Internal Processing (NoiseBlockAI)

NoiseBlockAI

Machine learning algorithm that suppresses non-speech noise (keyboard typing, paper shuffling, HVAC).

To Enable: Audio/Video > Audio > General Audio Settings > NoiseBlockAI.
Integrator Tip: If using an external DSP (Shure/QSC), DISABLE NoiseBlockAI and Acoustic Fence on the Poly codec to prevent double-processing artifacts.

Acoustic Fence

Creates a beam pattern that attenuates audio outside the defined zone. Critical for open workspaces.

  • Narrow (±12°): Focuses on a single person directly in front.
  • Medium (±30°): Good for a small table or huddle room.
  • Wide (±60°): Standard conference room coverage.

External Audio (DSP & IP Mics)

USB Audio (Shure/Biamp/QSC)

1. Connect DSP via USB-A.
2. Go to Audio/Video > Audio.
3. Enable "Enable USB Audio".
4. Verify input/output meters are active.

IP Microphones (LLN)

Connect Poly IP Table/Ceiling mics to the LLN port (G7500) or via PoE switch (G62). Ensure cabling uses CAT5e or better shielded cable.

Video Configuration & Tracking

Fine-tune Poly DirectorAI tracking modes and multi-camera behaviors.

Tracking Modes (DirectorAI)

Group Framing

Frames all participants in the room. Best for general meetings where context is needed.

Speaker Framing

Zooms in on the active speaker. Use for presentations or structured dialogue.

People Framing

Composite view (gallery) of up to 6 individuals. Best for hybrid equity.

Presenter Tracking

Follows a presenter in a specific zone. Requires E70/E60 camera.

Tuning Tracking

Settings Location: Audio/Video > Video Inputs

  • Tracking Speed: Set to Slow for executive rooms (less jarring), Fast for lively debates.
  • Framing Size: Tight for head-and-shoulders, Wide for upper body context.
  • DirectorAI Perimeter: Define the depth/width of the tracking zone to exclude glass walls or hallways.

Multi-Camera Setup (G7500/G62)

When using multiple cameras (e.g., E70s) for large rooms:

  1. Connect cameras via LLN (IP) or USB.
  2. In the Web UI, go to Audio/Video > Video Inputs.
  3. Assign cameras to roles (People vs Content).
  4. Use Camera Priority to determine which camera acts as the primary view when tracking is idle.

Mounting & Room Placement

Video Bar Placement

  • Height: Ideally eye-level with seated participants (approx 48").
    Placement too high (>7ft) causes "security cam" angles. Placement too low looks up noses.
  • Orientation:
    • Studio X30: Supports inverted mounting (below display).
    • Studio X50 / X52 / X70 / X72: Must be mounted UPRIGHT. Do not invert.

Codec Ventilation

Important: Poly G7500 and Studio G62 are modular codecs that generate heat.

Do not stack these units directly on top of other warm equipment. Ensure at least 1U of rack space or free airflow if mounted behind a display.

System Hardware Ports

Identify the ports on your system before beginning cabling.

Poly Studio X50 / X52 Ports

Back Panel Layout
  • 1 Power: Connects the supplied power supply.
  • 2 LAN: RJ-45 Ethernet port for network connection.
  • 3 HDMI In: Content sharing input (PC/Mac).
  • 4 HDMI Out 1 (Primary): Main room display (People/Content).
  • 5 HDMI Out 2 (Secondary): Optional second display.
  • 6 USB-A (x2): For peripherals (Touch monitors, USB cameras, Flash drives).
  • 7 USB-C: For PC connection (Device Mode).

Poly G7500 Ports

Back Panel Layout
  • LLN Link-Local Network (x3): RJ-45 ports specifically for IP peripherals (Poly IP Mics, Studio E70, TC8/TC10). Do not connect to LAN
  • LAN System LAN: Connection to corporate network.
  • HDCI Camera In: For EagleEye IV cameras.
  • Audio 3.5mm In/Out: For analog audio integration.

Cabling & Connections

1. Connecting Monitors (HDMI)

Poly systems support dual monitors. Monitor 1 is the primary display.

  • Connect the first monitor to HDMI Out 1. This will display the UI and main video.
  • (Optional) Connect the second monitor to HDMI Out 2. This is typically used for Content or spreading out far-end participants.
  • Note: Poly X30 only supports one monitor output.

2. Connecting Network (LAN)

Wired LAN is recommended for the best video quality.

Standard Connection: Plug a CAT5e/CAT6 cable from your network switch into the port labeled LAN on the codec.

3. Connecting the Touch Controller (TC10 / TC8)

Critical Network Rule
Do NOT plug the TC10/TC8 directly into the G7500 LLN ports. This is an unsupported configuration. The controller must be on the primary LAN to pair correctly with the system and access calendar services.

The touch controller is powered via PoE (Power over Ethernet).

Method A: Direct to Room Port

Connect the TC10/TC8 to a PoE-enabled wall jack on the same subnet as the video bar. This is the correct method for all systems (G7500, G62, Studio X).

4. Connecting Cameras & Audio (G7500 / G62)

For modular systems, you must connect peripherals manually.

  • USB Cameras (Studio E70/E60): Connect via USB-A port.
  • IP Cameras (Studio E70/E60): Connect to the LLN port (G7500) or via a PoE switch (G62).
  • IP Microphones: Connect via CAT cable to the LLN port.

Web Proxy Support

Poly VideoOS supports web proxies for signaling and HTTPS traffic.

Supported Protocols

  • Auto-Discovery: WPAD via DHCP Option 252.
  • PAC Files: Proxy Auto-Config files (.pac).
  • Manual: Static Proxy IP and Port.
  • Auth Methods: Basic, Digest, NTLMv2.

Configuration Path

Configure via System Web Interface:

Network > LAN Network > Web Proxy Settings
Best Practice: Ensure swupdate.lens.poly.com bypasses inspection or is allow-listed for software updates.

VLANs & LLDP

LLDP (Link Layer Discovery Protocol)

Enabled by default. The system advertises itself to the switch to automatically acquire VLAN tagging for Voice/Video if the switch is configured for LLDP-MED.

Manual VLAN Tagging

If LLDP is not used, you can manually set the VLAN ID in Network > LAN Network > LAN Options. Valid IDs: 1-4094.

Security & Hardening

Poly VideoOS includes enterprise-grade security features. Ensure these are configured for high-security environments.

Certificates & Encryption

Certificate Management

Poly systems ship with a self-signed certificate. For secure environments, you must install a certificate signed by your CA.

SCEP (Simple Certificate Enrollment Protocol):

Automates certificate issuance. Requires SCEP server URL and challenge password.

Admin Settings > Security > Certificates > SCEP
Manual Installation:

Supports .pem, .cer, .crt, .pfx, .p12 formats. Ensure you upload the full chain (Root CA > Intermediate > Device).

Admin Settings > Security > Certificates

Network Authentication (802.1x)

If your network ports are secured via 802.1x, configure the credentials before deploying the unit to the secured port.

  • EAP-TLS: Requires client certificate (highest security).
  • EAP-PEAPv0 (MSCHAPv2): Requires username/password.

Provider Specifics

Specific requirements and workflows for the major partner applications.

T

Microsoft Teams Rooms on Android

Critical: AOSP Migration

Starting with Poly VideoOS 4.5.2, all Microsoft Teams Rooms on Android devices will automatically migrate from Device Administrator to AOSP (Android Open Source Project) management in Intune.

  • Ensure your Intune environment allows AOSP enrollment.
  • Microsoft is deprecating the "Company Portal" method.
  • No action is required on the device itself; the update handles the OS transition.

Touch Controller Pairing

For Teams Rooms, the TC10/TC8 must be on the same subnet as the codec to pair. Pairing happens inside the Teams application, not just at the OS level.

  • Sign in to the Codec first.
  • A pairing code will appear on the TV.
  • Enter this code on the TC10/TC8 to complete pairing.
Z

Zoom Rooms

  • ZDM (Zoom Device Management): Poly devices can be managed directly via the Zoom portal (upgrades, alerts) once signed in.
  • Controller vs Scheduler: The Poly TC10 can be configured as a standalone Scheduler for outside the room, or a Controller for inside the room. This is selectable during the OOBE.
G

Google Meet

Supported on Studio X30, X50, X52, X70 (Not G7500).

  1. Boot the system and select Google Meet provider.
  2. A 6-character enrollment code will appear on the screen.
  3. Go to admin.google.com > Devices > Google Meet hardware.
  4. Enter the code to enroll the device into your Workspace domain.

Poly Lens Provisioning

Poly Lens provides centralized management, monitoring, and insight into your Poly devices.

Onboarding Methods

Cloud Pin Code

The simplest method for individual device onboarding.

  1. In the System Web Interface, go to Servers > Cloud > Provisioning.
  2. Copy the Device Pin.
  3. Log in to lens.poly.com.
  4. Go to Manage > Register Device and enter the Pin.

DHCP Auto-Discovery

Best for large scale deployments.

  • Configure DHCP Option 66 or 160 on your network server.
  • Point the URL to the Poly Lens provisioning server or your private instance.
  • Devices will automatically fetch credentials on boot.

Software Updates

Keep your system secure and feature-rich by maintaining the latest Poly VideoOS version.

Update Methods

Via Poly Lens

Log in to Poly Lens, select the device, and push the update remotely. This is the best method for fleets.

Via Web Interface

Go to General Settings > Device Management. Click "Check for Updates" to pull from the Poly server.

Via USB

Download the package from Poly Support. Place it on a FAT32 USB drive. Plug into the device.

Important
Never power off the device during a software update. The LED will blink amber/yellow during the process.

Troubleshooting & Reset

Common Issues

Controller Not Pairing

Ensure the TC10/TC8 and the Codec are on the same subnet. Check that multicast is enabled on the network. You can try manually pairing by entering the Codec IP address into the controller.

Black Screen on Monitor

Verify HDMI input on TV. Ensure the cable is plugged into HDMI Out 1 (the primary output). Try a different HDMI cable.

Factory Reset (Pinhole Method)

If the system is unresponsive or you need to clear all data, perform a hardware factory reset.

  1. Unplug the power cable from the device.
  2. Locate the reset pinhole (usually near the USB ports or on the bottom).
  3. Insert a straightened paperclip into the pinhole.
  4. While holding the reset button, plug the power cable back in.
  5. Continue holding the button for 15 seconds until the LED pulses amber (or blue/amber depending on model).
  6. Release the button. The system will wipe data and reboot to the out-of-box state.
